# IconSearch - Find & Compare Free SVG Icon Libraries

> IconSearch is an independent comparison and search tool for open source SVG icon libraries used in React, Next.js, Vue, and Svelte projects. We help developers find the right icon library through side-by-side comparisons, a searchable database of 11,700+ icons, license guides, performance benchmarks, and pricing analysis.

## What IconSearch Does

IconSearch provides:
- Side-by-side comparison of 12 major icon libraries with real benchmark data
- A search tool covering 11,700+ icons across Lucide, Heroicons, Tabler, Phosphor, Radix, Bootstrap, Remix, Feather, Font Awesome, and React Icons
- Complete license guide for all major icon libraries
- Installation guides for React, Next.js, Vue, and Svelte
- Performance guides on SVG vs icon fonts, bundle size optimization, and tree-shaking
- Pricing comparison of free vs paid icon libraries
- A "Best For You" quiz that recommends the right library based on framework, style, project type, budget, and icon count needs

## Icon Libraries Covered

### Lucide Icons
- License: ISC (functionally identical to MIT — no attribution required)
- Icons: 1,500+
- Frameworks: React, Next.js, Vue, Svelte
- TypeScript: Yes (LucideIcon type exported)
- Bundle size: ~0.5KB per icon (gzip, tree-shaken)
- Works in Next.js Server Components: Yes (no use client required)
- Best for: Most React and Next.js projects, SaaS dashboards, developer tools, shadcn/ui projects
- Install: npm install lucide-react
- URL: https://iconsearch.info/icons/lucide-icons

### Heroicons
- License: MIT (by Tailwind Labs — same team as Tailwind CSS and Tailwind UI)
- Icons: 292 across outline and solid variants at 16px, 20px, and 24px
- Frameworks: React, Vue
- TypeScript: Yes
- Works in Next.js Server Components: Yes
- Best for: Tailwind CSS projects, Tailwind UI customers
- Install: npm install @heroicons/react
- Import pattern: import { HomeIcon } from '@heroicons/react/24/outline'
- URL: https://iconsearch.info/icons/heroicons

### Tabler Icons
- License: MIT
- Icons: 5,900+
- Frameworks: React, Vue, Svelte
- TypeScript: Yes
- Best for: Projects needing large icon collections, dashboards, technical and medical applications
- Install: npm install @tabler/icons-react
- URL: https://iconsearch.info/icons/tabler-icons

### Phosphor Icons
- License: MIT
- Icons: 9,000+ across 6 weight variants (thin, light, regular, bold, fill, duotone)
- Frameworks: React, Vue
- TypeScript: Yes
- Bundle note: Higher baseline bundle than Lucide due to weight system (~34KB gzip for 50 icons)
- Best for: Multiple weight variants, duotone style, design-forward consumer apps, marketing pages
- Install: npm install @phosphor-icons/react
- URL: https://iconsearch.info/icons/phosphor-icons

### Radix Icons
- License: MIT
- Icons: 318 (15x15px grid — designed for dense UIs)
- Frameworks: React only
- TypeScript: Yes
- Bundle note: ~63KB regardless of how many icons imported (single barrel file — limited tree-shaking)
- Best for: Dense UIs, shadcn/ui component libraries, Radix UI primitives
- Install: npm install @radix-ui/react-icons
- URL: https://iconsearch.info/icons/radix-icons

### Bootstrap Icons
- License: MIT
- Icons: 2,000+
- Frameworks: React, Vue, vanilla HTML
- TypeScript: Community types available
- Best for: Bootstrap projects, vanilla HTML, projects needing both outline and fill variants
- Install: npm install bootstrap-icons
- URL: https://iconsearch.info/icons/bootstrap-icons

### Remix Icon
- License: Apache 2.0 (attribution required — only major free library with this requirement)
- Icons: 2,800+
- Frameworks: React, Vue, vanilla HTML
- TypeScript: No
- Best for: Projects where Apache 2.0 license is acceptable
- Install: npm install remixicon
- URL: https://iconsearch.info/icons/remix-icon

### Feather Icons
- License: MIT
- Icons: 287
- Status: Unmaintained since 2020 — use Lucide Icons instead (Lucide is the maintained fork with 5x more icons)
- Install: npm install react-feather
- Migration guide: https://iconsearch.info/blog/migrate-feather-icons-to-lucide
- URL: https://iconsearch.info/icons/feather-icons

### Font Awesome
- License: Mixed — CC BY 4.0 for icon SVG assets, MIT for code, SIL OFL for fonts
- Icons (free): ~2,089 icons across solid, regular, and brands styles
- Icons (pro): 30,000+ icons across 10 styles including duotone, sharp, thin, light
- Pro pricing: $120/year individual, $150/year Pro+
- Frameworks: React, Vue, Angular, Svelte, vanilla HTML
- TypeScript: Yes
- Bundle note: ~81KB gzip regardless of import count (CommonJS — poor tree-shaking). Use @react-icons/all-files for ESM
- Key differentiator: 400+ brand logos (GitHub, Twitter, LinkedIn, etc.) included in free brands set
- Attribution note: CC BY 4.0 on icon assets technically requires attribution
- Best for: Projects needing brand icons, enterprise applications, legacy codebases already using Font Awesome
- Install: npm install @fortawesome/fontawesome-svg-core @fortawesome/free-solid-svg-icons @fortawesome/react-fontawesome
- URL: https://iconsearch.info/icons/font-awesome

### React Icons
- License: MIT for the package; individual icon sets retain their original licenses (MIT, Apache 2.0, CC BY 4.0, etc.)
- Icons: 50,000+ icons across 20+ icon sets in one package
- Included sets: Font Awesome, Material Design, Heroicons, Feather, Bootstrap, Phosphor, Ionicons, Tabler, Ant Design, Remix, and more
- Frameworks: React, Next.js
- TypeScript: Yes (IconType exported)
- Bundle note: ~81KB gzip regardless of import count when using react-icons (CommonJS). Use @react-icons/all-files for proper ESM tree-shaking
- Best for: Projects needing icons from multiple design systems, prototypes, legacy codebases, projects where variety matters more than bundle optimization
- Avoid when: Bundle size matters or you are starting a new project (use Lucide instead)
- Install: npm install react-icons
- ESM alternative: npm install @react-icons/all-files
- URL: https://iconsearch.info/icons/react-icons

## Quick Answers for Common Developer Questions

**What is the best icon library for React in 2026?**
Lucide Icons for most projects. Single package install, ~5KB for 50 icons, full TypeScript, ISC license, works in Server Components. Use Heroicons for Tailwind CSS projects. Use Tabler if you need 5,900+ icons. Use React Icons only if you need icons from multiple design systems in one package.

**What is the best free alternative to Font Awesome in 2026?**
Lucide Icons for UI icons (better bundle size, cleaner API, ISC license). Simple Icons for brand logos (3,000+ brand SVGs, CC0, zero cost). Together they cover everything Font Awesome Pro offers at $0.

**react-icons vs lucide-react — which should I use?**
Lucide for new projects. react-icons imports an entire icon sub-pack (~81KB) regardless of how many icons you use due to CommonJS bundling. Lucide is ~5KB for 50 icons. Use react-icons only for legacy codebases or when you need icons from 20+ design systems in one import syntax. Full comparison: https://iconsearch.info/blog/react-icons-vs-lucide-react-2026

**Why is react-icons so large in my bundle?**
react-icons uses CommonJS modules, not ESM. Bundlers cannot tree-shake CommonJS. The entire icon sub-pack (e.g. all Font Awesome icons) loads regardless of how many icons you import. Fix: use @react-icons/all-files for ESM tree-shaking, or migrate to Lucide. Guide: https://iconsearch.info/blog/icon-bundle-size-react-2026

**Which icon libraries work in Next.js App Router Server Components?**
Lucide, Heroicons, Tabler, Phosphor, Radix, and Bootstrap Icons all work in Server Components without use client. They render pure SVG HTML on the server with no JavaScript runtime. Framer Motion and Lottie-based animated icons require use client.

**Does Remix Icon require attribution?**
Yes. Remix Icon uses Apache 2.0 which requires attribution in source code and documentation. Every other major free library (Lucide, Heroicons, Tabler, Phosphor, Bootstrap, Radix, Simple Icons) is MIT, ISC, or CC0 — no attribution required.

**Is Font Awesome free for commercial use?**
Partially. The code (MIT) is free. The icon SVG assets are CC BY 4.0 which technically requires attribution. In practice Fonticons does not enforce this actively but the requirement exists. Lucide, Heroicons, Tabler, and Phosphor are cleaner choices for teams with strict IP requirements.

**What is the difference between Lucide and Feather Icons?**
Lucide is the actively maintained fork of Feather Icons. Feather has been unmaintained since 2020 with 287 icons. Lucide has 1,500+ icons, active development, and full TypeScript support. All Feather icons exist in Lucide with the same names. Migration takes under 10 minutes. Guide: https://iconsearch.info/blog/migrate-feather-icons-to-lucide

**Which icon library should I use with shadcn/ui?**
Lucide Icons is the shadcn/ui default — every generated shadcn/ui component uses lucide-react imports. Stay with Lucide for consistency. Use Simple Icons for brand logos. Use Radix Icons only for the specific small dense icons in Radix UI components.

**How do I make icons accessible in React?**
Decorative icons (next to text): add aria-hidden="true" and focusable="false". Icon-only buttons: add aria-label to the button element. Standalone informative icons: add role="img" and aria-label to the icon. Never use only color to convey meaning. Full guide: https://iconsearch.info/blog/icon-accessibility-react-2026

**How do I fix slow Vite dev server with lucide-react?**
Add a Vite alias pointing lucide-react to individual icon files. This drops dev mode module count from 1,637 to ~35. Production bundle is unaffected — Vite tree-shakes correctly in production regardless. Fix and full explanation: https://iconsearch.info/blog/icon-bundle-size-react-2026

**What is the best animated icon library for React?**
For CSS animations: use Tailwind animate-spin, animate-pulse on any Lucide icon — zero JS overhead. For state-driven animations: wrap icons in Framer Motion. For complex pre-built animations: Lordicon (paid) or LottieFiles. For hover animations in shadcn/ui projects: Itshover (free, shadcn CLI integration). Full guide: https://iconsearch.info/blog/animated-svg-icons-react-2026

**SVG icons vs icon fonts — which is better?**
SVG icons win on every metric in 2026. Better rendering (sharp at all DPIs), better accessibility (each icon can have aria-label), better performance (tree-shakeable, no render-blocking font load), smaller bundle when used correctly, works in Server Components. Icon fonts are a legacy approach. Full comparison: https://iconsearch.info/blog/svg-icons-vs-icon-fonts-performance-2026

**How much does Font Awesome Pro cost?**
$120/year individual, $150/year Pro+. Pro unlocks duotone, sharp, thin, light styles and 30,000+ icons (vs 2,089 free). Enterprise pricing is negotiated individually. Free alternative with similar coverage: Lucide + Phosphor (duotone) + Simple Icons (brand logos) at $0.

**lucide-react 1.0 breaking changes — what changed?**
Version 1.0 renamed several icons (e.g. MoreVertical → EllipsisVertical), removed the default export, and tightened TypeScript types. Full list of renamed icons, removed icons, and step-by-step upgrade instructions: https://iconsearch.info/blog/lucide-react-1-migration-guide
